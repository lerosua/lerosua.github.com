周记 – 昃

昃， 这个字读“责”第四声，太阳偏西的意思。

工作/技术: 总是在快完结的时候才发现，某某地方还需要改善，某某地方又作了相应更改。有时总感觉剪不断，理还乱。代码开发永无终结，实现完功能，最后发现这样完成容易造成内存不足，于是又得重构，换一种方式。生命不断，代码不息。

地铁时间：看完了《赌场风云》，乔正初赢得了赌场，却输掉了身边所有的人，最终良心发现自首。身边人永远才是最大的成就，幸福的来源。准备停看剧集一段时间，看回书吧，找了韩寒的《1988，我想和这个世界谈谈》来看。一如既往熟练地玩弄文字的风格，不过一些隐寓太过于浅显，着痕迹，不过这一切都正合我心水。看他的书永远不会闷啊。

时事：香港继续反国教，拒绝洗脑的游行。越来越多人加入。这一关失守，则沦为大陆一般，谎言之地。香港要hold住。

总结：这星期平平淡淡，静如流水。

运程：★

期待：iphone5的发布

周记 – 盈

流水账：周日从北京飞回来。以前人上京赶考，一走三个月，现在时代进步，交通发达，上京也只需要三个小时。但是我觉得还是远远不够。交通应该更发达，有生之年能赶上广州去北京只需要三十分钟吗，有点悬。
周一去考了驾试的科目一，有惊无险，取得90分合格。然后今天去学车，却被告知没车，还是条漫长的路。
工作上的事情嘛，北京回来之后，app作相应更改，预料中事了。但是上级们又想尽快发布，矛盾啊。于是员工们加班吧。还好这星期不用加，好好休息下。然后开始死命加？唉，IT就是这样吧。

视频：《怒火街头2》看完了，结局一般般，TVB越来越堕落了。于是找回以前的剧集看，开始看《赌场风云》。《造王者》里，董昭终于露出了狐狸真面目，真正是大奸似忠，然后像之两面讨好，城府极深，本来就是政治家的典范嘛。而余靖这角色其实愚忠，正直，冲动，能生存到现在根本靠的是主角光环。比如之前得罪漕帮完全是找死十几回了，次次都要靠个严三娘来救场，完全不是其解决问题的…

电影：这星期精彩了，连续看了几部新电影，最值得一提的当然是《奥德克巴莱》，如果文明是让我们卑躬屈膝，那我们就让你们见识野蛮的骄傲。此金句已不用再提。其中令人深思，各种现代化设备并未给人们带来多少幸福，反而让人们意识到自己多么贫穷。真正的文明，不应该让人们有所选择吗，是过现代化生活，还是回到狩猎时代更简单明了。当然对于剧里强大的日军火力，三百勇士明知抗争是死，仍为之。是以不肯曲膝求存，有时候，尊严即生命，失去尊严，同行尸走肉有何分别。人就一辈子，如昙花一现，壮丽一时也未不是选择。这，让人想起了杨佳。

随想：梓童的一个同学通过了公立学校教师招聘，然后终于分配到了学校，是我们母校。此位同学由大专毕业，继而考到本科证，再考上公立学校，一步一个脚印，也算励志啊。然而我想的是，又回到自己的母校，与曾经是自己的老师的人做同事，那是一种怎样的感觉啊。很早之前，我希望的职业也是老师，但希望是大学里的。总希望上课轻松，有寒暑假那么爽吧。

周记 – 月

技术/工作：上周末连续加班，接着周一周二继续开发。周三赶飞机。以前是坐南航，这次是国航，似乎飞机更多大，而且是两层的。但是飞机晚点这个事儿呢，习惯中的习惯了。出了机场，老板的司机来接，回到酒店也花了差不多一小时。路上发现司机是不系安全带的，很多的士司机也不系 … 这事儿 …

然后一直就是在酒店和公司之间徘徊。第一天工作量还挺大，第二天还好。主要是面临测试状态，多人使用，立马就发现许多bug。于是立马改，又立马装。iphone装测试挺麻烦的，首先要把iphone加进开发者设备里，然后在开发证书里加上此设备为测试机，然后才能安装。于是昨天在公司一下子有五台iphone摆在面前让我装的阵势。用iphone的女性用户偏多啊。晚上老板请团队吃饭。当老板的人普遍能侃。我当个倾听者好了。

今天工作结束得早。于是又聚齐了京城四少一起吃自助。想起离开京城也正好一年。但是见到他们也并没有久别的感觉。因为网上天天见到他们。大家聚在一起，又象往常一样胡天胡地扯开了谈。我一如既往地对jyf1987的想法嗤之以鄙，然后他又华丽丽地藐视我。yetist也一向谦虚厚道，xihels则发福了不少，当了领导果然不同了，呵呵。之后还一起去围观了一下朝鲜使馆，鄙视一下金家。这烂 朝鲜能有多少外交事务，可却占了那么大块地,浪费。

明天飞机回广州了。终于不用住旅馆了。

视频:晚上继续追看《造王者》，好吧，董昭是忍辱负重，千夫所指而无间道。但说起视频，还是说下视频聊天吧。因为出差，于是得和梓童联系。电话本来是快速的联系方式。但是网络发达的今天，起码也得视频聊天吧，面对着面，无障碍感嘛。梓童在家有ipad，我有iphone，本来facetime是最方便的事情。但似乎因为两个设备都用同一账号，不好自己连自己。于是改用QQ视频，效果还行。而且几乎免费，所以像传统线路通话什么的怎么还没有被淘汰呢… 这不合理嘛

时事:周日应该发生了几大城市反日游行示威吧，全天下没有比这更傻B的事情了，你就算要反日，但砸的还是中国人的车，中国人的店。某party天天日你你没意见，日本人日你一下你就跳出来了，还是一个英国人总结的好，中国人只有在安全时勇敢，免费时慷慨。脑子被洗了真可怕。

其它：北京地下人行道晚上都是演唱厅啊，各路文艺青年都在这试音。

周记 – 日

随便说说

8月一号，电信的套餐终于转成了上网版，也就是有400M流量的。中国电信客服很喜欢关爱用户，经常会打电话来问用户，比如问为什么不多用打多点电话，上多点网啊。我说过很多次，我的那个电信号码完全是电信营业厅流氓捆绑宽带时一起开通的。送的一部电阻屏的“智能手机”，落后时代五年。到后来我才搞明白，开的套餐是聊天版，120M流量，360分钟通话，管我用不用得了这么多，每月在银行卡照扣80￥。我非常非常之不爽。也后悔当时一时冲动为了带宽去开通。后来公司配了部4s当开发机，正好是电信版。用上了真正的智能机，流量当然多多不够用了。去研究电信的套餐，才明了同样价格还有个上网版套餐。也就是400M流量的。于是打10000号去客服让换。打了两次电话，结果返回的结果是不同的，一个说能转，一个死活说不能转。于是我让能转的帮转，投诉那个不能转的。
今天再去查套餐，果然转成了上网版，这电信，管理真混乱。当国外在建设G为单位的带宽时，天朝这还在为争多几十k而计算。那些所谓的砖家怎么有老脸提什么大国崛起啊。

然后新华社今天的一篇文章，归纳出新时代的黑五类，不禁让人联想，共同富裕的第二步将要执行似的，就是把不富裕的全都消灭掉，从而实现共同富裕。未来的形势不容乐观啊。

前阵子参加的同学聚餐，才得知某同学和某同学终于也分手了。大学时代我们班自产自销了三对情侣。感觉这比例也挺高的吧，他们分别于，毕业前分手，毕业就分手，毕业后分手。毕业后分手那对，其实我挺看好他们的，或许终究是熬不过异地相思吧。也许不到老，你也不知道，谁是真正会陪你终生的人。

下班回家的路上，仍旧看到了一个人硊在地上拼命地磕头，口里喊着，谢谢大家。旁边摆着一个睡着的老妇人。我走过的时候瞟了地上的纸两眼，大约是他是个司机，出了个车祸赔了钱，现在老母亲病重无钱医治什么的。先不论真假，但是我已经心安理得地把结论推给政府，要怨怨政府的不作为吧。生在天朝都是不幸人，只是有些人更加不幸。

周记 – 宙

往古来今谓之宙,这宙表示时间。

时事：这部分就太精彩兼太悲催了。北京一场大雨，折射出这个外表光鲜，内里腐败的天朝多少问题啊。失败的城市建设，集政治，经济，文化三为一体，各种资源如此优越，却也经不起一场雨的考验，还有脸宣传成百年一遇的雨。某新华社报纸几十年前的形容用之才为贴切，一党独裁，遍地是灾。大雨造成的经常损失瞬间就能计算出来，而人员的伤亡却永远是个谜。听之闻之为之一叹。这要放古代，天子脚下，怎么也得下个罪已诏吧，不然也开粮仓救济灾民啊。还有脸提出捐款。捐你妹啊。把每年捐给那些独裁国家的款收回来再说吧。生得计划，死得随机，真是一点也没说错，怎么能想到在天朝公路上开着个车也能淹死…这还真冤啊，这完全不科学啊。完全可以收录进天朝奇妙物语里。

Git服务器搭建

因需要私人项目，需要跟朋友协助，因此需要代码管理仓库。github本来应该很好的，但我暂时没美刀支付渠道，曾想过dropbox+git本地管理，但发现dropbox已经禁用这种使用方式了。后来找了个bitbucket.org的，可支持私有项目，只是协助成员限制在五个人以内，这也没问题，但后来发现在天朝的烂网络下，太慢了。最后老犁借了个vps给我，让我在上面自己建一个。于是有了折腾。但发现其实还是非常简单的。

网上已经非常多的教程了。这里就我也啰嗦再贴一次，主要参考了

Git服务器Gitosis安装设置 , 使用gitosis的来配置管理的git服务器端

1. 准备环境：

老犁的vps装的是ubuntu，因此用apt-get安装以下工具 

sudo apt-get install openssh-server openssh-client git-core python-setuptools
然后建个git的用户 
sudo useradd git
sudo passwd git
创建git仓库存储目录 
sudo mkdir /home/git/repositories
设置git仓库权限 
sudo chown git:git /home/git/repositories
sudo chmod 755 /home/git/repositories
初始化全局设置 
git config --global user.name "git"
git config --global user.email "git@server"
2.安装git服务器

这里选用的是gitosis，教程上面全是让git clone一个出来的，这里我其实好奇怪，为啥不用apt-get直接装库里有的呢，我自己就试了下安装，但发现它在下一步gitosis-init的时候出错，难道仓库里的gitosis坏了没人管啊，于是我也只有去clone一个出来，但网上 git clone git://eagain.net/gitosis.git 这个地址似乎失效了，还好找到了替代。 

git clone https://github.com/res0nat0r/gitosis.git
安装需要用到python-setuptools 
cd gitosis
sudo python setup.py install
3.配置git服务器

身边linuxer，接触到服务器这方面，多少也早就有自己的ssh key了吧，没有的话ssh-keygen -t rsa自己生成一个，这里不细说。把自己的公钥~/.ssh/id_rsa.pub 上传到服务器。 

scp ~/.ssh/id_rsa.pub 你的名字@你的服务器:/tmp
然后用公钥去配置服务器 
 sudo -H -u git gitosis-init < /tmp/id_rsa.pub
Initialized empty Git repository in /home/repositories/gitosis-admin.git/
Reinitialized existing Git repository in /home/repositories/gitosis-admin.git/
后面两行是输出。(命令中的小于号似乎显示不了…)

然后是设置post-update脚本权限，其实我看这的权限本来已经是755了，但网上都有这一步，是保险吧。 

chmod 755 /home/git/repositories/gitosis-admin.git/hooks/post-update
至此服务器方面的操作完成。

4.管理git服务器

git服务器的一个好处是它可以通过git回客户端来配置管理，现在回到自己的客户端，也就是上传了公钥的那台机器，因为这里有能解它的私钥啊。
clone出git服务器的配置。 

git@你的服务器:gitosis-admin.git
取回来后进到gitosis-admin目录，会看到目录结构很简单，一个gitosis.conf配置文件，和一个keydir目录，里面放着之前导入的那个公钥。
配置文件类似 
cat gitosis.conf 
[gitosis]

[group gitosis-admin]
writable = gitosis-admin
members = leros@MacBook-Pro.local 这里有个建议，如无必要不要去修改管理者的密钥名啥了，之前我试过改，然后就导致 ERROR:gitosis.serve.main:Repository read access denied 的错误。另外，如果members里的名称和keydir里的公钥文件名不一致也会导致这个错误，所谓一致是指 members里是xxx，则公钥文件名y对应为xxx.pub

5.添加新项目及新加协助成员

那么，现在我们差不多可以正式工作了，建立一个新项目，app-test,邀请成员lerosuz加入协助。那么我们就必须在服务器配置文件里加入 

cat gitosis.conf 
[gitosis]

[group gitosis-admin]
writable = gitosis-admin
members = leros@MacBook-Pro.local
[group cyclone-team]
writable = app-test
members = leros@MacBook-Pro.local lerosuz 如上所示，我们新建了一个组叫cyclone-team，组员是管理员leros他新成员lerosuz，管理员获取到lerosuz的公钥,并将之命名为lerosuz.pub，加入到keydir中，然后提交修改至服务器，于是就会生效了。命令大概是： 

git add keydir/lerosuz.pub
git add gitosis.conf
git ci -m "添加lerosuz协助"
git push
以上是设置了新项目的读写权限。现在可以正式建立新项目了。
还是在笔记本终端中： 
mkdir /tmp/app-test
cd /tmp/app-test/
git init
touch testfile
git remote add origin git@你的服务器:app-test.git
git add testfile
git ci -m "inital"
git push origin master:refs/heads/master
完成以上命令，则项目就会传到服务器上，新成员lerosuz就可以clone项目下来协助了。 
git clone git@你的服务器:app-test.git
当然，没有加进公钥的人是无法clone下来的。

PS.以上的git命令我用的是简写，请确保你的主目录下的.gitconfig有以下内容: 

 cat ~/.gitconfig 
[alias]
	st = status
	ci = commit
	co = checkout
	br = branch
	dc = dcommit
	rb = rebase
enjoy~
