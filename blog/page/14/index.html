折腾owncloud小记

| Comments

注：本篇不是教程，只是个人折腾记述。

Dropbox大家都知道的了，但毕竟是私有公司产品。那么对应的开源产品就是owncloud,它可以让你搭建自己的云，完全自己控制，有VPS的同学千万不要错过，比如偶。

例行公式，给自己的云建个域名吧，比如  cloud.xxx.org , xxx.org则是你的域名，这里建立了一个coud.xxx.org的子域名。并让它指向一个地址，比如VPS上的 /var/www/owncloud

然后去owncloud的官网下载最新的压缩包, 但我安装5.0的版本，却始终有问题，最后下载了4.5.8版本才成功。各位按需下载。

下面的步骤基本上跟官方指南一致了。这里是Linux服务器上的安装指南地址http://doc.owncloud.org/server/5.0/adminmanual/installation/installationlinux.html

『折腾环境,VPS-debian』
而下面是我折腾的步骤，当然我已经按要求装好了各种需要的依赖包了，主要是PHP方面的依赖，PHP是我自己编译的，后来发现它还需要PHP的zip扩展。PHP 5.4之后的版本源码带了zip扩展，因为需要重新编译它，加上–enable-zip才可打开这个支持。 

tar xvfj  owncloud-4.5.8.tar.bz2
chown www-data:www-data -R owncloud
sudo mv owncloud /var/www/
owncloud可以跟apache2协同工作，但我VPS上用Nginx,于是贴上Nginx的配置（官网上有标准的配置文件，加了ssl的配置） 
# Now comes the main configuration for ownCloud 5
server {
  listen 80;

server_name cloud.xxxx.org;
root /var/www/owncloud;
index index.php;
client_max_body_size 1000M; # set maximum upload size

# deny direct access
location ~ ^/(data|config|.ht|db_structure.xml|README) {
deny all;
}

# default try order
location / {

rewrite ^/.well-known/host-meta /public.php?service=host-meta last;
rewrite ^/.well-known/carddav /remote.php/carddav/ redirect;
rewrite ^/.well-known/caldav /remote.php/caldav/ redirect;
rewrite ^/apps/calendar/caldav.php /remote.php/caldav/ last;
rewrite ^/apps/contacts/carddav.php /remote.php/carddav/ last;
rewrite ^/apps/([^/]*)/(.*.(css|php))$ /index.php?app=$1&getfile=$2 last;
rewrite ^/remote/(.*) /remote.php/$1 last;
try_files $uri $uri/ @webdav;
}

# owncloud WebDAV
location @webdav {
fastcgi_split_path_info ^(.+.php)(/.*)$;
fastcgi_pass 127.0.0.1:9000; # or use php-fpm with: “unix:/var/run/php-fpm/php-fpm.sock;”
fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
#fastcgi_param HTTPS on;
include fastcgi_params;
}

# enable php
location ~ .php$ {
try_files $uri = 404;
fastcgi_pass 127.0.0.1:9000; # or use php-fpm with: “unix:/var/run/php-fpm/php-fpm.sock;”
fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
#fastcgi_param HTTPS on;
fastcgi_param htaccessWorking true;
include fastcgi_params;
}
}

之后就是让nginx读取这个配置

/etc/init.d/nginx reload
现在可以在浏览器中打开 http://cloud.xxx.org 进行配置了。
基本上输入用户名和密码，默认使用sqlite数据库，即可创建好环境。
这里我遇到一个问题说一下，就是之后在管理页面，我创建用户始终不成功，后来发现，原来在osx下的chrome不成功，改成safari则可以…

最后，去下载一个客户端同步工具吧，地址在http://owncloud.org/install/
根据自己平台选择客户端，连接那地址也是之前的http://cloud.xxx.org

如果你跟我一样有使用goagent代理的习惯，注意把客户端中的代理关掉，因为它默认使用系统代理在我这造成了不能同步的问题。
所有都成功之后，把文件丢到你的owncloud目录里，则自动同步到云去了。

android端有个owncloud的apk，收费的，但功能不怎么好，并且我没试过用它上传文件成功过…

周记 – 雨

| Comments

工作：星期天开始重新投入工作，这一版本主要解决合并订单的问题，于是订单预览和订单详情页面都得重写。主要都是繁琐的细节，所谓慢工出细活，没有时间，陶练不出细节。

生活：一年房租合约到期，房东想把房子拿回去自已住。然则又是要找房子了，到时搬家又是一个麻烦，又要给中介赚一笔。暂时打算在同一小区里找，希望租金市场别涨得太厉害，给条活路吧。

电影：这周属于电影周。连续看了四部电影。『十二生肖』，寻找回一点飞鹰的影子。前面一大段都挺好的，当然在荒岛里好几个桥段却也复制以前的片断，比如法国女拿机枪扫射。最后一段JC突然去保卫龙头的爱国心的转变没有过程，过于突兀。总体还是不错。

然后是『一代宗师』，或许是我抱有的心态不同，不应该以功夫片或武打片的期望去看这片。这是文艺片。从梁朝伟的叶问开始在一边旁白的时候就应该知道。然后就是觉得一堆习武之人太有文化了，妓院也太文艺了，像网友说的，一个个像油画般站着。那哪是妓院，那是国家艺术中心吧… 至于功夫，我觉得还是去看『叶问』吧，慢镜头功夫像十年前张艺谋的『英雄』一样。另外，这个是宫二传，叶问打了最大的酱油，张震第二，张震演的叫啥我也不知道，另外看到小沈阳的脸就觉得违和… 听说原来有四个小时，被迫减成这样，我在想，四个小时搞个上下集嘛，花都花了这么多年时间拍了，最终搞成这样大家都心疼啊。

之后是看西游系列了，看了个动画片『金箍棒传奇』，我觉得以后要有一种电影类型叫吐槽电影吧。其实这片演绎得不错，对玉帝等天神的颠覆嘲讽到位，就是如果换个画风，就肯定神不少，现在看来像是flash动画。而『西游降魔篇』，怎么说呢，之前摆的期望过高了吧。其实上笑点并不是太多，但一些剧情效果是不错的，高老庄那段，孙悟空大战三大驱魔人那段，都非常好。黄渤的文戏那也很不错，符合人物心理。后来得知似乎从悟空传演绎过来，原来编剧里有今何在。但是嘛，有朱玉在前，这里的爱情片断相对来说，就不怎么来电了，因为基本上是重复大话的模式，而且还用了大话的经典曲目，『一生所爱』。其实听着『一生所爱』，所能想到的绝对是至尊宝和紫霞了。所以在这里再怎么唱，也只不过复述了别人的传奇。或许这桥段对新生代有效，因为他们没经历过大话时代。『一生所爱』在大话里用得才好，最后孙悟空孤独去赶上取经队伍，音乐响起，粤语的唱腔也更好听。实际上看完『西游降魔篇』，我真想再回去看看『大话西游』，重温那段悲伤的演绎。苦海，翻起爱浪…

地铁时间：终于看完了『中国底层访谈录』，我觉得这应该是天朝版的『悲惨世界』才对。讲述上一辈各种因缘际会被政治左右了的人生。讲述了监狱里的事情，会明白，监狱出来的人怎么会是改过自新的人呢，那里是恶的集中地，各种折磨折腾，也有各种阶层，把大社会一下子浓缩到一个小监牢里，于是放大各种恶行，唯一没有的，就是尊严。采访的多数上访的人员，几乎都是冤案难平，总是一个晴天霹雳就把生活给搅乱了。越要平反，则生活越困窘。你会明白，努力不一定有收获，正义从来没被申张。只要走入反贪官不反皇帝的逻辑，最后都是穷图末路，如果明白了要反对的是整个制度，那又是立马被毙。所以会得出一个结论，不能寻求正义，才能苟活。莲花能出淤泥而不染，那是因为它已高高在上，是谁也不能在淤泥中而不染吧，因此，不能清洗掉淤泥，则只能远离它。

运程：★

期待：希望能找到个好的房源，买不起，也快租不起了。房市这泡沫啥时候破啊…

没怎么走亲戚。感觉今年冷清多了，鞭炮声也只是断断续续的。这小镇有如天朝，有钱的都往外搬。只有白天有些许游人来游荡一番，到了夜晚留下一地的垃圾，被风卷着在空中飘荡。

但还是要逛老婆娘家，她那边亲戚倒很活跃。不过中国人堆在一块，共同的嗜好也就只剩下赌了。女的打麻将，男的打牌。红色的，青色的老毛在桌子上穿梭交错。我不赌，不吸烟，不喝酒，也不说话。静静地坐在一旁边。或许自己太过于怪异了。

终于给家里换了台电视，39寸液晶电视。之前那个松下电视用了17年吧，现在天气潮湿，要开半小时才能缓缓清晰过来，有时候图像又会慢慢缩小，最后变成一条线，只有重开或者大力拍拍它才能恢复。说服父母换这个电视也废了很多唇舌。老人家觉得还能用的，就将就，不要浪费钱。如今换了大屏幕电视，起码看得清了，也省电了。唯一看着那个机顶盒，我还是要骂一下，这反人类的烂货。

视频：又再次看了『生死拳』，主要看武打场面。再次对它的剧情感到纠结，于是终于搜索到原版的『黑拳』，还原了感觉。我了个操广电。电影面目全非。原版暗黑，现实，醒世。而被广电剪了小jj后果然只能看看武打，剧情一塌糊涂。不过罗守耀导的戏都挺喜欢的，有银河范，又一贯的暗黑系风格。最近两年似乎没怎么导新戏了，难道钱赔光了，不敢玩了，这可不要啊。

时事：所谓新政，新一轮的百花齐放，结果来了个程爱华案，东厂走狗连年都不让人好好过，急于立功献媚，再立文字狱。俄国天降陨石，据说一百年前，也是天降陨石，然后三年后大清灭亡，不过靠天相安慰人心，玄乎幻乎。

运程：★

期待：长远一点，找到真正要做的事业，然后，所有东西都纷致踏来。

放假：在广州呆了一天多，然后坐顺风车回老家了。回到家第一件事就是去开通网络，于是跑去电信营业厅，排了一会儿队，然后被告知需要开户人身份证原件复印件等，真是一堆麻烦，不过又被告知可以打电信服务电话开通服务。这才回家用电话搞定，提供一个开户人证件号码即可。这才知道自己真是傻，跑来跑去不如一个电话方便。家里又看到固话的话费单，每月准扣26元，家里接听的多，不多打电话。固话现在还扣这么多钱，真不值，我真想替他们换成移动手机卡，固定10元，接听免费。但考虑到父母换电话号码的成本，算了。

视频：主要看『刀剑神域』，题材不错，但剧情有些跳脱，后来了解了一些内情才明白，原来是小说改编，而后又扩充内容。男主和女主的爱情渲染得不错，如果直接在14集结束即可完美。偏偏后面加了一半拯救女主的剧情，实际上攻略妹子，结果成了后宫动画…

阅读：持续在看『中国底层访谈录』，由不同人的访谈整理而成，有个上访户，被整得很惨，然而却寄希望于新政权，期望来个钦差大差，包青天之类，帮他们平反。实际上上访户都是这类想法吧，皇上是好的，只是被奸臣蒙蔽了。思想限制境界，大概是这个意思吧。但然而又期望他们能出个啥觉悟呢。又有个朝圣者，讲的是信西藏佛教的事，里面对汉族信佛的批评真的很贴切，说汉族人信佛只为升官发财，有事所求时抱佛脚，这种不诚心，我们都知道。 而另一些人在做不了官，发不财，看破红尘，心灰意冷时才信佛，所谓看破红尘，实际上却又只是一种逃避方式，为佛所不耻。而西藏人从出生就沐浴在佛宗信仰里，最好的子女都会送去庙宇，视之理所当然，视之荣耀，无所谓出世入世。当然我讲的很简单，但我也深刻理解了汉族人，真的不懂宗教。

过年：回到家的那时，妈妈跟我唠叨，要我快生个孩子。也的确是，我长大了，爸爸今年要六十了，家里四个大人，过年没点气氛，要有小孩才有年味吧。今年是时候认真考虑这个问题吧。祖国未统一不能再是借口了。

运程：★

期待：过年拜年啥的我最不期待了，现在又光是给人家发红包，又没小孩收红包。

结语：新年快乐，岁月静好！

工作：虽说之前发了年前最重要最磨心的一版本，不过趁着放假前，还是要再发一个小版本更新。为年后的一些功能做准备。然后调整假期安排，明天加班，周三放，希望周三能放了，关键在于周一能否顺利发布了。

折腾：在osx上装了一下android开发环境，eclipse+android sdk等，不过android sdk的一些域名都是被墙的，下载起来特别麻烦。但装好了也没下文了，java实在有点搞不懂兼不想搞。然后是编译gaeproxy，主要是去掉广告栏和换一个顺眼点的图标。结果发现它内置的HTTPS选项是连到私人服务器的，要验证，第三方不可用，安全的理由。结果没有那个选项，twitter官方客户端根本没法用。一是为了这个客户端用原生的，一是放弃这个客户端用自己编译的。结果还是用自己编译的算了，上twitter还是别的方法嘛。

运程：★

期待：下星期终于放假回家过年了，希望年终资金给力。

解决gitosis问题一则

| Comments

最近github被墙，看来自建的git服务器还是有派上用场的时候。自建git服务器请看这里 但一段时间没理，再回去git pull竟然出现了问题:

Traceback (most recent call last):  
  File "/usr/bin/gitosis-serve", line 5, in   
    from pkg_resources import load_entry_point  
  File "/usr/lib/python2.6/dist-packages/pkg_resources.py", line 2671, in   
    working_set.require(__requires__)  
  File "/usr/lib/python2.6/dist-packages/pkg_resources.py", line 654, in require  
    needed = self.resolve(parse_requirements(requirements))  
  File "/usr/lib/python2.6/dist-packages/pkg_resources.py", line 552, in resolve  
    raise DistributionNotFound(req)  
pkg_resources.DistributionNotFound: gitosis==0.2

此时深深理解菜博士黑python的心情啊。
通过Google终于在一个日本人的博客找到解决办法。竟然是python版本依赖问题…
把 

/usr/bin/gitosis-init /usr/bin/gitosis-run-hook /usr/bin/gitosis-serve
里的 
#!/usr/bin/python
改成 
#!/usr/bin/python2.5
即可。

另一个问题是，没找到git服务器的进程，好像不需要啥命令启动这个服务。
