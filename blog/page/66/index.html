Watting for the End of Copyright

| Comments

今天再看韩寒的一篇博客！得知高晓松要他在《三重门》里引用它的一首歌的歌词去掉。可能还会打官司！我看了之后可真是晕啊！后来找高晓松的博客看了看。噢，原来高晓松和陆川是哥们！

什么文坛之争啊，就是两拨人在闹！先是白烨，然后陆天明父子自己蹦出来，裙带关系扯到现在的高晓松。至于韩寒那有什么人我就不清楚了。不过据他们的说法是有一大拨人匿名，隐身地跑去白烨等人的博客去骂。这些人也是有失光明的！偶也支持韩寒。不过偶也不会做跑到那些人的博客骂人不留名的事！那不是韩的作风，也不是偶的作风。偶也没当韩是偶像。偶是就这个理来直挺韩寒的！

白烨的学术造假给这趟讨伐韩寒的声音蒙混过去了！他们就拉了亲朋戚友一起来讨伐所谓的八十年代后。现在又拿所谓的版权说事！就法律上是通的。在道德来说这人心胸倒是有点窄。偶不是文坛人，偶关注的IT业，这个行业里有一拨人，他们把自己所写的代码贡献出来，几乎是无私地贡献出来，而已还允许你自由地复制，修改，和再发布（当然要保留原作者信息之类,注：偶说的不专业，具体请参考GPL v2）。这里实事求是，追求真理。团结互助。

如果文坛就上面那些人，偶很庆幸偶小时候的梦想没有成真（偶小时候也是想过做一个作家的）。偶的偶像是Rechard.M.Stallman. 偶等待版权终结的时代！

上课

毕业班了，但还有课，那一定是很少人上的！偶们班就特别是地少了！当然有一部分人是找到工作上班去了！但还是有一些是没找到工作的！不过他们也很少去上课的！不是留在宿舍打游戏就是在睡觉！课堂上的出勤率特别低！

网络还虚无吗？

记得以前有句话，在互联网上没人知道你是只狗！可是在今天，这句话就不真实了！上了网就真的没人知道你是谁了吗？不一定吧！ 大多数人网聊，可是到了一定的时候就会暴露出现实身份的一些信息。只要根据这些信息，是有可能追踪到你个人的！另外，每个人都有一个ip地址！根据这个地址也很容易可以追踪到你的位置！因此，不要以为在互联网上做些坏事就没人知道了！网络上你的痕迹会你现实中你的痕迹更加真实！

前一阵子说是有个手机实名制，那大家的手机方面的信息无疑是更容易给监控了！刚才看新闻说是网络游戏也搞实名制！我看以后可能会有个ip实名制！每个人一出生的时候除了领取身分证，还得到一个ip哩！
还好我不玩网游的。手机没必要的话也不要用了吧！呵呵！
我们的zhenfu有点像专制的父母，深怕自己的孩子学坏了，目光是一刻不肯离开孩子，要把孩子的一切都掌握在手中！

其实适当地放一放手，不是更好嘛！

关于最近韩寒新闻的一些乱侃!

昨天一网友发给我一个博客地址说是韩寒的博客。偶很有兴趣地去看了！晕！原来是新浪的博客。这点令我的firefox无故崩溃了！mozilla也不能幸免！这让偶挺恨这些不标准的网页的！

无奈之下只有用opera来看了！看了才知道最近韩寒和白烨搞上口水战了！热闹非凡啊！看看他的利索的文笔，有醒神的效果啊！想想在高中的时候，无意中看到他的那篇《杯中窥人》，就不禁为他的文笔而折服！后来更是买了他的第一本小说《三重门》来看！当时觉得这样一个同龄人能做到这样很厉害！当时偶对高考的制度也是深恶痛绝的！因此就很关注关于他的事！后来知道他退学了！然后就不知消息了！然后偶也自个上大学了！没有再听到他的消息！

后来倒是出了不少冒充韩寒作品的小说面世！有一本还骗到了我，叫什么《新三重门》封面写着是韩寒新著。那时我还天真地以为真的是新著！不过那之后我就认清了类似金庸新著之类的书的真面目。

上了大学以后听说韩寒又出了新书，《像少年啦飞驰》，看过之后也没多大感觉了！他的书大抵都反应虚无！至少我看到的是！不过还是有些句子偶比较喜欢的，如他一朋友自杀了。警察问他为什么他会自杀，他回答：文学青年嘛，都这样！然后警察又问他，为什么样你不去自杀？他回答，我不是文学青年！虽然我已经好多年没看小说了！但从韩寒的小说中还是能得到一点乐趣的！

关于现在韩寒和白烨的争吵，当然不是什么八十后和什么文坛的争斗什么。这样说的人纯粹是意淫，即意淫了文坛还意淫了整整一代人！（这话是韩寒说的吧？偶不记得了）。反正两个人都不足以代表什么！互联网上就是喜欢这么瞎搞，陈凯哥告胡艾的事也是这么搞出来的！也是啊，你不出个新闻我吃完饭和朋友侃什么啊！明白了这个道理后我也释然！看个新闻而已嘛！偶能发什么感叹呢！

不过我还是很想看看韩寒以后的事情，毕竟偶和他是同龄人，也是高中时关注至今！看看不同的人走的不同的路，是件有趣的事！

白烨是谁我之前是一点也不知道的！不过现在韩白之争后偶搜索了一下，大概了解了他的光辉事迹(策划《911生死婚礼—我的情爱自传》事件)！
不禁汗颜啊！偶借陈导的一句话给他：我觉得做人不能无耻到这种地步！

Fetchmail收gmail遇到的一个问题

| Comments

这些天因为不满evolution的内存表现于是想换一个邮件工具！选择上了mutt!

实际为网上的经典组合 mutt + fetchmail + esmtp

偶这里不做什么介绍了！网上相关的教程多得是：不过有趣的是，搜索到的有好大一些文章是直接抄王垠的。关于fvwm，tex，mutt,的他的入门文章的确不错！当初偶看他的fvwm介绍入门使用Fvwm,想不到今天仍是看他的mutt文章来使用Mutt。
不知道的可以点这里了解一下什么是mutt ，因为别人已经写得这么详细了！偶也不重复轮子了！

如果是要收发gmail的邮件的话，可以看看linuxsir上的精华文文章，用fetchmail接收和发送gmail邮箱的邮件

偶要说的就是关于这篇文章的一点补充！偶照着它做了，不过仍然是收不了gmail的邮件！如果你也是的话，可以看看下面的提示。

在做完所有步骤后可以先运行 fetchmail -c来测试下是否设置正确！

偶出现了下面这个错误：

TLS library problem: 17552:error:14090086:SSL routines:SSL3_GET_SERVER_CERTIFICATE:certificate verify failed:s3_clnt.c:894

解决的方法，找google啊！这里找到答案 。主要是在 .fetchmail 文件里去掉
sslcertck sslcertpath . 写成是下面格式就成了:

poll pop.gmail.com proto POP3 and options no dns
user ‘name@gmail.com’ there with password ‘password’ is ‘name’ here options ssl

是什么原理我没认真看，最近没太多时间了。希望能帮到一些人！
