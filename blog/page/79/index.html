Fvwm环境下openoffice界面设置

| Comments

Fvwm环境下openoffice界面设置

先把openoffice.org-gnome装上，这个是基础支持。然后设置好你的.gtkrc-2.0，选好你的主题！

跟着只要欺骗ooo,让它以为偶们是运行在gnome环境下的！运行下面命令：

xprop -root -format GNOME_SM_PROXY 8i -set GNOME_SM_PROXY 1

然后运行ooo, 界面出来了吧！呵呵！

这个方法对acroread也有效。
献给所有使用Fvwm的fans，这使偶们的界面更统一接近完美了。 :)

设置代理上Google

| Comments

我不知道偶学校为什么要封google,真的是因为怕我们上那些反动的网站吗？但为了那一小撮人而让广大的校友不能用google的服务这样又公平吗？你别和偶说百度。那个是消遣的东西，对于找一些实际有用的东西是没用的，起码对于编程这方面的东西。现在的实际情况是国外的知识资源比国内的丰富多了！有很多次偶为了不能用google找资料气得半死。学校这网，真的只是开给人搞娱乐的而已吧。

偶不得已只有用代理上了！

偶说下在linux的方法吧，网上已经有很多的帖子了，大多出自www.linuxsir.org的吧，偶也是那看的！简单地把几个步骤说完它吧！

1. 装个debian吧（说笑而已，偶是基于debian做的！

2. 安装 tor, privoxy, tsocks
apt-get install tor privoxy tsocks

在 /etc/tsocks.conf中设置以下行：

server = 127.0.0.1
server_type = 5
server_port = 9050

把“ forward-socks4a / localhost:9050 . ”写进privoxy的配置文件/etc/privoxy/config里，端口号后面是有个点的！别忘了！

3. 启动tor和privoxy服务

/etc/init.d/tor start && /etc/init.d/privoxy start

在firefox里将代理地址设置为: localhost:8118

4.可以上google了！偶的gmail也可以用了！:)

MP3 to OGG

| Comments

前天没有关机是因为要把电脑上二千首的MP3格式的歌转换成OGG模式的！至于为什么要这样做，可能是无聊的原因吧！

mp3转成ogg在linux是有个命令可以直接转的，叫mp32ogg。实际上是个perl脚本来的！ 里面的内容我也没细看，但转出来的ogg却比原来的mp3体积更大！因此我找了个用mplayer编码来转换。原理是先用mplayer将mp3还原成wav格式，然后再用oggenc压缩成ogg。命令如下:

mplayer -vo null -vc dummy -ao pcm:waveheader $i && oggenc -r audiodump.wav -o $i

命令完后就将此文件的后缀改成ogg就行了！

这样转换的ogg比原来的mp3小多了！但速度是问题啊，所以我才转了一天一夜。后来一看最后所有的文件加起来偶空出了将近 2 G 的空间！

但后来偶才发现一个比较晕的问题，这样的转换mp3的idv标签 会丢失 ！于是偶二千首歌的标签全空了！

算，事已至此，后悔已是徒劳！其实做这件事之前就应该想好各种情况的！ 所以做什么事之前深思熟虑是应该的也是必须的！

Evolution出现的小问题（关于corba)

| Comments

今天遇到问题了，evolution突然不能启动了，Blogtk也发生了段错误！真是奇怪了！

evoluton的启动错误是

(evolution:26100): Bonobo-Activation-WARNING **: Strange exception (IDL:omg.org/CORBA/COMM_FAILURE:1.0) from active server registration

上网找了下关于这个corba的错误，有几个不同的案例，但语焉不详，或不了了之！当初偶找的都是中文页面。谁叫偶这上不了google呢！百度，真的只能用来找无聊的信息的！

然后我用root用户启动evolution，竟然可以启动，那就是说是偶用户怕某些设置问题了！于是偶先是看下权限的设置有没有更改。因为偶以前也试过权限的设置更改后程序出现莫名的错误！看了下还是没问题，后来在 /tmp下看到一些目录.其中有 orbit-leros ,再进去看看，发现两个相关的文件：

bonobo-activation-register.lock
bonobo-activation-server-ior

于是把它删除了，再启动evolution,成功启动了！ 虽然是解决了，但还是不明白为什么突然会出现这样的情况。开机太久了？ 我昨天没关机，现在开机时间有一天过十二小时了！

刚才用ps看了一下进程才发现关于 bonobo-activation-server 的进程已经启动了三四个了！我想这大概就是问题的所在了，因为这个进程太多而无法进行通信吧！看来在开发问题的所在时调查进程的状态也是很重要的！

但BloGtk的问题还是没解决，段错误一般是内存错误，它什么错误提示都没有。难道是我装了什么东西和它冲突了？等待解决！没了BloGTK,现在写博客也有些麻烦！

转换系统编码

| Comments

昨天看到一个编码转换的命令，是convmv，可以批量转换文件名的编码。于是就想把自己的系统从gb系列转到utf8。我系统上用的是GB2312。主要是我自己目录里的中文需要转换。一直以来用gb都有些小的暗伤，好多时遇到一些超出GB2312编码的字时就会有点小麻烦！本来用GB18030就可以的啦。不过这个国标有些大！虽解决了字的问题，但仍然是有些兼容的问题。gtk+系统用的编码默认就是UTF8。在使用Linux时常常要接触到这个编码。因UTF8和GB系列是不兼容的！这就是麻烦的来源！这也是一直以来未下定决心转编码的原因。

不过今天还是突然决定要转成UTF8的！ 文件内的编码可以用iconv来转，好在不是很多文件的内容要我转，我的vim可以自动识别编码，因此转不转也问题不大！但一些程序在UTF8的环境下会出些问题。最麻烦的是xmms和gmplayer。 我的几G的音乐本来是用GB2312编码的文件名，这个可以用convmv来批量转换：

convmv -f cp936 -t utf8 –notest -r /music

但mp3里的id3v2的那些标签就是一个麻烦！它仍是gb编码，xmms在utf8的情况下已经不能正常显示它了！现在唯一的方法只有把id3v2禁用了吧！ 现在改用rhythmbox好了！不过感觉还是没有xmms好！也许哪天我把机上的mp3全部转成ogg好了！那样就不用这样烦恼了！

gmplayer在utf8下已经乱码了，菜单一塌胡涂。因为偶还用的是gtk1版本的gmlayer，是自己编码的！我想现在装个gtk+版的gmplayer挺好！
